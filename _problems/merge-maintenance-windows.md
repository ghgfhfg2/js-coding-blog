여러 점검 구간이 주어질 때, 겹치거나 바로 이어지는 시간을 합쳐 전체 점검 시간을 구하는 `solution` 함수를 작성하세요.

## 문제 설명
서버 점검 기록이 2차원 배열 `windows`로 주어집니다.
각 원소는 `[start, end]` 형태이며, `start`분부터 `end`분까지 점검이 잡혀 있다는 뜻입니다.

여러 점검 구간이 서로 **겹칠 수 있고**, 끝나는 시각과 다음 점검 시작 시각이 같은 경우처럼 **바로 이어질 수도 있습니다**.
이럴 때는 실제로는 끊기지 않은 하나의 긴 점검 시간으로 봅니다.

모든 점검 구간을 적절히 합친 뒤, **전체 점검이 진행된 총 시간 길이**를 반환하세요.

예를 들어 `[[1, 4], [2, 6], [8, 10]]`이라면:

- `[1, 4]`와 `[2, 6]`은 겹치므로 `[1, 6]`으로 합쳐집니다.
- `[8, 10]`은 따로 유지됩니다.
- 전체 길이는 `(6 - 1) + (10 - 8) = 7` 입니다.

## 제한사항
- `1 <= windows.length <= 100000`
- 각 구간은 `[start, end]` 형태입니다.
- `0 <= start < end <= 1000000000`
- 반환값은 모든 점검 구간을 합친 총 길이입니다.
- `[1, 3]`과 `[3, 5]`처럼 끝과 시작이 맞닿아 있으면 하나의 연속 구간으로 합칩니다.

## 예시
- 입력: `[[1, 4], [2, 6], [8, 10]]` → 출력: `7`
- 입력: `[[5, 7], [1, 3], [3, 5]]` → 출력: `6`
- 입력: `[[2, 8], [3, 4], [5, 7], [8, 11]]` → 출력: `9`

## 힌트
- 구간이 섞여 들어오므로 먼저 시작 시간을 기준으로 정렬해 보세요.
- 현재 합치고 있는 구간의 끝보다 다음 구간의 시작이 작거나 같다면 이어붙일 수 있습니다.
- 합칠 수 없는 순간이 오면, 지금까지의 길이를 더하고 새 구간을 시작하면 됩니다.

## 해설
이 문제의 핵심은 **정렬 후 구간 병합(interval merge)** 입니다.

입력 순서 그대로는 어떤 구간이 서로 겹치는지 한눈에 보기 어렵습니다.
그래서 먼저 `start` 기준으로 오름차순 정렬합니다.

정렬한 뒤에는 왼쪽부터 보면서 현재 병합 중인 구간 `[curStart, curEnd]`를 유지하면 됩니다.

### 풀이 아이디어
1. `windows`를 시작 시각 기준으로 정렬합니다.
2. 첫 구간을 현재 구간으로 잡습니다.
3. 다음 구간 `[start, end]`를 확인합니다.
   - `start <= curEnd` 이면 겹치거나 바로 이어지는 것이므로
     `curEnd = Math.max(curEnd, end)`로 확장합니다.
   - 그렇지 않으면 현재 구간 길이 `curEnd - curStart`를 정답에 더하고,
     새 구간을 현재 구간으로 바꿉니다.
4. 순회가 끝난 뒤 마지막 현재 구간 길이도 더합니다.

### 왜 정렬이 필요할까?
예를 들어 `[[5, 7], [1, 3], [3, 5]]`가 그대로 들어오면,
앞에서부터만 봐서는 뒤쪽 구간이 사실 전부 이어진다는 점을 바로 처리하기 어렵습니다.

정렬하면 `[[1, 3], [3, 5], [5, 7]]`가 되고,
이제는 순서대로 보면서 하나의 긴 구간 `[1, 7]`로 자연스럽게 합칠 수 있습니다.

### 함정 포인트
- **겹치는 경우만** 합치는 게 아니라, **끝과 시작이 같은 경우도 합쳐야** 합니다.
  - 즉 조건은 `start < curEnd`가 아니라 `start <= curEnd`입니다.
- 입력이 이미 정렬되어 있다고 가정하면 안 됩니다.
- 구간이 완전히 포함되는 경우도 있습니다.
  - 예: `[2, 8]` 안에 `[3, 4]`, `[5, 7]`이 들어 있어도 잘 처리되어야 합니다.

### 복잡도
- 정렬: `O(n log n)`
- 병합 순회: `O(n)`
- 전체: `O(n log n)`

구간 개수가 많아도 정렬 후 한 번만 훑으면 되기 때문에 충분히 효율적입니다.
