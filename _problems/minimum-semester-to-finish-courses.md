과목 수 `n`과 선수과목 정보 `prerequisites`가 주어집니다.

각 원소 `[a, b]`는 **과목 `a`를 먼저 끝내야 과목 `b`를 다음 학기부터 들을 수 있다**는 뜻입니다.
한 학기에는 조건만 만족하면 여러 과목을 동시에 수강할 수 있습니다.

모든 과목 `1..n`을 끝내기 위해 필요한 **최소 학기 수**를 반환하는 `solution` 함수를 작성하세요.
만약 선수과목 관계에 사이클이 있어 모든 과목을 끝낼 수 없다면 `-1`을 반환하세요.

## 제한사항
- `1 <= n <= 100000`
- `0 <= prerequisites.length <= 200000`
- `prerequisites[i] = [a, b]`
- `1 <= a, b <= n`
- `a !== b`
- 같은 선수과목 쌍이 중복으로 들어오지 않는다고 가정합니다.
- 과목 `a`를 끝낸 학기와 같은 학기에는 `b`를 들을 수 없고, **다음 학기부터만** 수강할 수 있습니다.

## 예시
- 입력: `n = 4`, `prerequisites = [[1, 3], [2, 3], [3, 4]]` → 출력: `3`
  - 1학기: `1, 2`
  - 2학기: `3`
  - 3학기: `4`
- 입력: `n = 5`, `prerequisites = []` → 출력: `1`
  - 모든 과목을 바로 첫 학기에 들을 수 있습니다.
- 입력: `n = 4`, `prerequisites = [[1, 2], [2, 3], [3, 1]]` → 출력: `-1`
  - 순환 의존이 있으므로 모든 과목을 끝낼 수 없습니다.

## 힌트
- 선수과목 관계는 방향 그래프로 볼 수 있습니다.
- 지금 들을 수 있는 과목은 **진입 차수(indegree)가 0인 노드**입니다.
- 어떤 과목의 최소 학기는, 그 과목으로 들어오는 선수 경로들 중 **가장 늦게 끝나는 경로 다음 학기**가 됩니다.

## 해설
핵심은 이 문제를 **DAG(사이클이 없으면)** 위의 스케줄링 문제로 보는 것입니다.

한 학기에 여러 과목을 들을 수 있으므로, 각 과목은 “가능한 가장 빠른 학기”에만 배치하면 전체 최소 학기 수가 됩니다.
즉 각 과목에 대해 다음 값을 구하면 됩니다.

- `semester[x] = 과목 x를 가장 빨리 들을 수 있는 학기`

### 1) 그래프와 진입 차수 만들기
`[a, b]`는 `a -> b` 간선입니다.
- `a`를 끝내야 `b`를 들을 수 있으므로
- `b`의 진입 차수를 1 증가시킵니다.

진입 차수가 0인 과목은 선수과목이 없으므로 **1학기**에 바로 들을 수 있습니다.

### 2) 위상 정렬하면서 최소 학기 갱신하기
큐에서 진입 차수 0인 과목 `cur`를 꺼내고, `cur`에서 갈 수 있는 다음 과목 `next`를 확인합니다.

`next`는 `cur`를 끝낸 **다음 학기부터** 가능하므로:

```js
semester[next] = Math.max(semester[next], semester[cur] + 1)
```

왜 `max`가 필요할까요?
`next`에 선수과목이 여러 개 있을 수 있기 때문입니다.
예를 들어 어떤 과목이 2학기짜리 경로와 4학기짜리 경로를 동시에 기다린다면, 결국 **더 늦는 쪽**을 다 끝내야 시작할 수 있습니다.
즉 이 문제는 각 노드까지의 **최장 경로 길이**를 구하는 것과 같습니다.

### 3) 사이클 판별
위상 정렬로 처리한 과목 수가 `n`보다 작다면, 어느 시점부터도 진입 차수 0인 과목이 나오지 않는다는 뜻이고 그래프 안에 사이클이 있습니다.
이 경우 정답은 `-1`입니다.

### 4) 복잡도
- 그래프 구성: `O(n + m)`
- 위상 정렬: `O(n + m)`
- 전체: `O(n + m)`

여기서 `m = prerequisites.length` 입니다.

배열 길이가 큰 입력에서도 충분히 통과할 수 있는 정석 풀이입니다.
