회의 시간 구간 배열 `meetings`가 주어질 때, **서로 겹치지 않게 선택할 수 있는 회의의 최대 개수**를 반환하는 `solution` 함수를 작성하세요.

각 회의는 `[start, end]` 형태이며, 어떤 회의가 끝나는 시각과 다른 회의가 시작하는 시각이 같다면 **겹치지 않는 것**으로 봅니다.

## 제한사항
- `1 <= meetings.length <= 100000`
- 각 원소는 `[start, end]` 형태의 정수 배열입니다.
- `0 <= start <= end <= 1000000`
- 끝나는 시각과 다음 회의 시작 시각이 같으면 둘 다 선택할 수 있습니다.
- 반환값은 선택 가능한 최대 회의 수입니다.

## 예시
- 입력:
  ```js
  [
    [1, 4],
    [2, 3],
    [3, 5],
    [6, 7]
  ]
  ```
  출력: `3`
  - `[2, 3]`, `[3, 5]`, `[6, 7]`처럼 고르면 3개를 선택할 수 있습니다.

- 입력:
  ```js
  [
    [1, 2],
    [2, 3],
    [3, 4],
    [1, 3]
  ]
  ```
  출력: `3`
  - 끝과 시작이 같은 경우는 허용되므로 `[1, 2]`, `[2, 3]`, `[3, 4]`를 모두 고를 수 있습니다.

- 입력:
  ```js
  [
    [5, 5],
    [5, 6],
    [6, 6]
  ]
  ```
  출력: `3`
  - 길이가 0인 회의도 있을 수 있으며, 조건만 맞으면 함께 선택할 수 있습니다.

## 힌트
- 회의를 시작 시간이 빠른 순서로만 고르면 최적이 아닐 수 있습니다.
- 어떤 회의를 고른 뒤에는 **다음 선택의 여지를 가장 많이 남기는 회의**가 유리합니다.
- 그래서 끝나는 시각이 빠른 회의부터 보는 greedy 전략을 떠올려 보세요.

## 해설
이 문제는 가능한 모든 회의 조합을 직접 만들어 보면 경우의 수가 너무 많아집니다.

핵심은 **끝나는 시각이 가장 빠른 회의를 먼저 선택하는 것이 항상 유리하다**는 점입니다.

왜 그럴까요?
끝이 늦는 회의를 먼저 고르면 그 뒤에 넣을 수 있는 회의 수가 줄어듭니다.
반대로 끝이 빠른 회의를 고르면 이후 시간대가 더 많이 남아서, 다음 회의를 이어 붙일 가능성이 커집니다.

예를 들어 `[1, 4]`와 `[2, 3]` 중 하나만 골라야 한다면, `[2, 3]`을 고르는 편이 뒤에 `[3, 5]`나 `[3, 4]` 같은 회의를 더 붙이기 좋습니다.

### 풀이 흐름
1. 회의를 **끝나는 시각 오름차순**으로 정렬합니다.
2. 끝나는 시각이 같다면, 시작 시각이 빠른 순으로 정렬해도 됩니다.
3. `lastEnd`를 현재 마지막으로 선택한 회의의 종료 시각으로 둡니다.
4. 정렬된 회의를 앞에서부터 보면서,
   - `start >= lastEnd` 이면 현재 회의를 선택합니다.
   - 선택했다면 개수를 1 늘리고 `lastEnd = end`로 갱신합니다.
5. 순회가 끝나면 선택한 개수가 정답입니다.

### 왜 greedy가 맞는가?
어떤 시점에서 선택 가능한 회의들 중 끝나는 시각이 가장 빠른 회의를 골라도, 이후에 선택 가능한 회의 수가 줄어들지 않습니다.
오히려 남는 시간이 가장 많아지므로 다음 선택에 가장 유리합니다.
이런 성질 덕분에 지역적으로 좋은 선택이 전체 최적해로 이어집니다.

### 시간 복잡도
- 정렬: `O(n log n)`
- 한 번 순회: `O(n)`
- 전체: `O(n log n)`

이 문제의 포인트는 **구간을 전부 비교하려 하지 말고, 끝나는 시각 기준으로 정렬한 뒤 가장 빨리 비워지는 선택을 쌓아 가는 greedy 감각**입니다.
