하루 매출 기록이 주어질 때, 누적 매출이 목표 이상이 되는 가장 빠른 날짜를 구하는 `solution` 함수를 작성하세요.

## 문제 설명
정수 `target`과 정수 배열 `sales`가 주어집니다.
`sales[i]`는 `i + 1`번째 날의 매출입니다.

첫날부터 순서대로 매출을 누적했을 때, **누적 매출이 처음으로 `target` 이상이 되는 날짜 번호**를 반환하세요.
그런 날짜가 끝까지 없으면 `-1`을 반환합니다.

예를 들어 `target = 10`, `sales = [2, 3, 1, 4]`라면:

- 1일차 누적 매출: `2`
- 2일차 누적 매출: `5`
- 3일차 누적 매출: `6`
- 4일차 누적 매출: `10`

처음으로 목표 이상이 되는 날은 4일차이므로 정답은 `4`입니다.

## 제한사항
- `target`은 1 이상 1,000,000 이하의 정수입니다.
- `sales`의 길이는 0 이상 100,000 이하입니다.
- `sales`의 각 원소는 0 이상 10,000 이하의 정수입니다.
- 날짜 번호는 1부터 시작합니다.
- 목표를 끝까지 넘기지 못하면 `-1`을 반환합니다.

## 예시
- 입력: `10, [2, 3, 1, 4]` → 출력: `4`
- 입력: `7, [7, 0, 2]` → 출력: `1`
- 입력: `15, [5, 4, 3]` → 출력: `-1`
- 입력: `6, []` → 출력: `-1`

## 힌트
- 배열을 앞에서부터 보면서 누적 합을 계속 더해 보세요.
- 누적 합이 `target` 이상이 되는 순간 바로 날짜를 반환하면 됩니다.
- 끝까지 한 번도 넘기지 못했다면 `-1`입니다.

## 해설
이 문제의 핵심은 **누적 합을 앞에서부터 관리하는 것**입니다.

1. `sum` 같은 변수를 0으로 시작합니다.
2. `sales`를 0번 인덱스부터 차례대로 순회합니다.
3. 현재 날의 매출을 `sum`에 더합니다.
4. 더한 뒤 `sum >= target`이면, 그날이 처음 목표를 넘긴 날이므로 `i + 1`을 반환합니다.
5. 반복이 끝날 때까지 조건을 만족하지 못하면 `-1`을 반환합니다.

예를 들어 `target = 5`, `sales = [1, 1, 1, 1, 1]`이라면:

- 1일차 누적: `1`
- 2일차 누적: `2`
- 3일차 누적: `3`
- 4일차 누적: `4`
- 5일차 누적: `5`

따라서 정답은 `5`입니다.

이 문제는 모든 날의 누적 합 배열을 따로 만들 필요 없이, 합계를 하나만 유지하면서 한 번 순회하면 해결할 수 있는 기본적인 **prefix sum / running sum** 문제입니다.
