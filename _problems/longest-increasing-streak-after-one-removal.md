정수 배열 `nums`가 주어질 때, **원소를 최대 한 개까지 제거한 뒤** 만들 수 있는 가장 긴 **엄격 증가 연속 구간**의 길이를 반환하는 `solution` 함수를 작성하세요.

여기서 엄격 증가 연속 구간이란, 구간 안의 모든 인접한 두 원소에 대해 앞 원소가 뒤 원소보다 작은 구간을 뜻합니다.

## 제한사항
- `1 <= nums.length <= 100000`
- `nums[i]`는 정수입니다.
- 원소는 제거하지 않아도 됩니다.
- 제거 후에도 구간은 **연속한 인덱스들로 이루어져야** 합니다.
- 같은 값이 이어지면 증가 구간으로 보지 않습니다. 즉, `a < b`여야 합니다.

## 예시
- 입력: `[1, 2, 5, 3, 4, 6]` → 출력: `5`
  - `5`를 제거하면 `[1, 2, 3, 4, 6]`이 되어 길이 `5`의 증가 구간을 만들 수 있습니다.
- 입력: `[1, 2, 3, 4]` → 출력: `4`
  - 이미 전체가 증가 구간이므로 제거할 필요가 없습니다.
- 입력: `[5, 4, 3, 2]` → 출력: `1`
  - 하나를 제거해도 길이 2 이상의 증가 구간은 만들 수 없습니다.
- 입력: `[2, 2, 3, 4, 5]` → 출력: `4`
  - 앞의 `2` 하나를 제거하면 `[2, 3, 4, 5]`가 되어 길이 `4`를 만들 수 있습니다.

## 힌트
- 어떤 위치를 제거한다고 가정했을 때, 그 왼쪽에서 끝나는 증가 구간 길이와 오른쪽에서 시작하는 증가 구간 길이를 빠르게 알아내면 좋습니다.
- `nums[i]`를 제거했을 때 `nums[i - 1] < nums[i + 1]`가 성립하면, 양옆 증가 구간을 이어 붙일 수 있습니다.
- 모든 위치를 제거 후보로 보더라도, 각 위치의 정보를 미리 계산해 두면 전체를 선형 시간에 처리할 수 있습니다.

## 해설
이 문제를 단순하게 풀면, 각 위치를 하나씩 제거해 보고 매번 가장 긴 증가 구간을 다시 계산해야 해서 비효율적입니다.

핵심은 **각 위치까지 이어지는 증가 길이**와 **각 위치부터 시작하는 증가 길이**를 미리 계산하는 것입니다.

### 1) 왼쪽에서 끝나는 증가 길이
`left[i]`를 `i`에서 끝나는 가장 긴 엄격 증가 연속 구간 길이라고 합시다.

- 항상 최소 1입니다.
- 만약 `nums[i - 1] < nums[i]`라면,
  `left[i] = left[i - 1] + 1`
- 아니면 `left[i] = 1`

### 2) 오른쪽에서 시작하는 증가 길이
`right[i]`를 `i`에서 시작하는 가장 긴 엄격 증가 연속 구간 길이라고 합시다.

- 항상 최소 1입니다.
- 만약 `nums[i] < nums[i + 1]`라면,
  `right[i] = right[i + 1] + 1`
- 아니면 `right[i] = 1`

### 3) 제거하지 않는 경우도 먼저 고려
답은 적어도 배열 안에서 원래 존재하는 가장 긴 증가 구간 길이 이상입니다.
즉 `left` 배열의 최댓값을 기본 정답 후보로 잡을 수 있습니다.

### 4) 각 위치를 제거해 보기
이제 각 인덱스 `i`를 제거한다고 생각해 봅시다.

- 맨 앞 원소를 제거하면 `right[1]`이 후보가 됩니다.
- 맨 뒤 원소를 제거하면 `left[n - 2]`가 후보가 됩니다.
- 중간 원소를 제거하는 경우:
  - 왼쪽 구간 길이: `left[i - 1]`
  - 오른쪽 구간 길이: `right[i + 1]`
  - 그리고 **`nums[i - 1] < nums[i + 1]`** 이면 두 구간을 이어 붙일 수 있습니다.
    - 후보 길이 = `left[i - 1] + right[i + 1]`
  - 그렇지 않으면 둘을 연결할 수 없으므로, 둘 중 큰 쪽만 유지됩니다.

예를 들어 `[1, 2, 5, 3, 4, 6]`에서 `5`를 제거하면
- 왼쪽의 `[1, 2]`
- 오른쪽의 `[3, 4, 6]`
을 이어 붙일 수 있고, 실제로 `2 < 3`이므로 길이 `5`가 됩니다.

반면 `[1, 4, 2, 3]`에서 `4`를 제거하면 연결 가능하지만,
다른 위치를 제거했을 때는 양옆 값의 대소관계가 맞지 않아 이어 붙이지 못할 수도 있습니다.

### 시간 복잡도
- `left` 계산: `O(n)`
- `right` 계산: `O(n)`
- 제거 후보 확인: `O(n)`

따라서 전체 시간 복잡도는 `O(n)`이고, `n`이 커도 충분히 처리할 수 있습니다.

이 문제의 포인트는 단순히 "하나 지워 본다"가 아니라, **삭제 전후의 증가 구간 정보를 양쪽에서 미리 모아 두고 연결 가능성만 판단하는 것**입니다.
