물류 허브가 `1..n`번으로 번호 매겨진 도시들과 도로 정보 `roads`가 주어집니다.

각 도로는 `[a, b, risk]` 형태이며, 도시 `a`와 `b`를 양방향으로 연결하고 이 도로를 지날 때의 **위험도**가 `risk`라는 뜻입니다.

배송 경로의 전체 위험도는 **경로에 포함된 도로들 중 가장 큰 위험도 1개**로 정의합니다.
즉, 여러 도로를 지나더라도 합계를 구하는 것이 아니라, 지나간 도로 중 가장 위험한 구간의 값이 그 경로의 점수입니다.

출발 도시 `start`에서 도착 도시 `end`까지 이동할 때,
이 전체 위험도를 **최소화한 값**을 반환하는 `solution` 함수를 작성하세요.
도착할 수 없으면 `-1`을 반환하세요.

## 제한사항
- `1 <= n <= 100000`
- `0 <= roads.length <= 200000`
- `roads[i] = [a, b, risk]`
- `1 <= a, b <= n`
- `a !== b`
- `1 <= risk <= 1000000000`
- 같은 두 도시 사이에 여러 도로가 있을 수 있습니다.
- `1 <= start, end <= n`
- `start === end`일 수도 있습니다.

## 예시
- 입력: `n = 5`, `roads = [[1, 2, 4], [2, 5, 3], [1, 3, 6], [3, 4, 2], [4, 5, 5], [2, 3, 7]]`, `start = 1`, `end = 5` → 출력: `4`
  - `1 -> 2 -> 5` 경로의 위험도는 `max(4, 3) = 4`
  - `1 -> 3 -> 4 -> 5` 경로의 위험도는 `max(6, 2, 5) = 6`
  - 따라서 최소 가능한 값은 `4`입니다.

- 입력: `n = 4`, `roads = [[1, 2, 8], [2, 4, 8], [1, 3, 3], [3, 4, 10]]`, `start = 1`, `end = 4` → 출력: `8`
  - 합계만 보면 `1 -> 3 -> 4`가 작아 보일 수 있지만, 이 문제는 최대 구간 위험도를 봅니다.
  - 두 후보의 위험도는 각각 `8`, `10`이므로 정답은 `8`입니다.

- 입력: `n = 3`, `roads = [[1, 2, 7]]`, `start = 1`, `end = 3` → 출력: `-1`
  - 도착 도시로 갈 수 있는 경로가 없습니다.

## 힌트
- 일반 최단거리처럼 “지금까지의 합”을 저장하면 안 됩니다.
- 어떤 도시까지 가는 경로의 비용은 **지나온 간선들의 최댓값**입니다.
- 현재까지의 최대 위험도가 가장 작은 도시부터 확정해 나가면 됩니다.

## 해설
이 문제는 겉보기에는 최단거리처럼 보이지만, 경로 비용 정의가 다릅니다.

보통 다익스트라는:

- `새 비용 = 현재 비용 + 간선 가중치`

를 사용합니다.

하지만 여기서는 경로 점수가 “합”이 아니라 “지나온 간선 중 최댓값”이므로,
현재 도시 `cur`까지의 최적 비용이 `dist[cur]`일 때 간선 위험도가 `w`인 다음 도시 `next`의 새 후보 비용은 다음과 같습니다.

```js
candidate = Math.max(dist[cur], w)
```

즉, 지금까지 경로에서 가장 위험했던 구간과 새로 추가하는 도로 위험도 중 더 큰 값이 새로운 경로 점수가 됩니다.

### 1) 그래프 만들기
각 도로 `[a, b, risk]`는 양방향이므로 인접 리스트에 둘 다 넣습니다.

### 2) `dist[x]`의 의미
`dist[x]`를

- `start`에서 `x`까지 갈 때 가능한 경로들 중
- **최대 구간 위험도**가 가장 작은 값

으로 정의합니다.

초기값은 모두 `Infinity`, 시작점은 `0`입니다.
`start === end`면 아무 도로도 지나지 않으므로 답은 `0`입니다.

### 3) 왜 다익스트라가 통하나?
어떤 도시를 우선순위 큐에서 꺼냈을 때, 그 도시의 현재 비용이 아직 처리되지 않은 후보들 중 가장 작다면,
그보다 더 좋은 경로가 나중에 나타날 수 없습니다.

이유는 새 경로를 붙일 때 비용이

- `max(현재까지 비용, 새 간선 위험도)`

형태로만 커지거나 유지되지, 더 작아지지 않기 때문입니다.
이 단조성이 있어서 다익스트라 방식의 “가장 좋은 후보부터 확정” 전략이 그대로 성립합니다.

### 4) 알고리즘 흐름
1. 인접 리스트를 만든다.
2. `dist[start] = 0`으로 둔다.
3. 최소 힙(우선순위 큐)에 `[0, start]`를 넣는다.
4. 가장 작은 비용 후보를 꺼낸다.
5. 이미 더 좋은 값이 기록되어 있으면 버린다.
6. 인접한 도시마다 `candidate = max(cost, risk)`를 계산한다.
7. `candidate < dist[next]`이면 갱신하고 힙에 넣는다.
8. 끝까지 갔을 때 `dist[end]`가 여전히 `Infinity`면 `-1`, 아니면 그 값을 반환한다.

### 5) 복잡도
- 그래프 구성: `O(roads.length)`
- 다익스트라: `O((n + roads.length) log n)` 수준

도로 수가 큰 입력에서도 충분히 사용할 수 있는 풀이입니다.

핵심은 **최단거리의 합 최소화**가 아니라 **병목 구간의 위험도 최소화**라는 점입니다.
이 차이 때문에 점화식이 `+`가 아니라 `max`로 바뀝니다.
