정수로 이루어진 2차원 배열 `board`가 주어질 때, **모든 칸의 숫자가 같은 정사각형 부분 배열** 중에서 가장 큰 것의 한 변 길이를 반환하는 `solution` 함수를 작성하세요.

예를 들어 `2`만으로 이루어진 `3 x 3` 정사각형이 있다면 정답은 `3`입니다.

## 제한사항
- `1 <= board.length <= 500`
- `1 <= board[i].length <= 500`
- 모든 행의 길이는 같습니다.
- `board[i][j]`는 정수입니다.
- 반환값은 만들 수 있는 가장 큰 같은 숫자 정사각형의 **한 변 길이**입니다.

## 예시
- 입력:
  ```js
  [
    [1, 1, 1],
    [1, 1, 1],
    [1, 1, 1]
  ]
  ```
  출력: `3`
  - 전체 보드가 같은 숫자이므로 한 변 길이 3의 정사각형을 만들 수 있습니다.

- 입력:
  ```js
  [
    [1, 2, 2, 2],
    [3, 2, 2, 2],
    [4, 2, 2, 2],
    [5, 6, 7, 8]
  ]
  ```
  출력: `3`
  - 오른쪽 위 영역에 `2`로 이루어진 `3 x 3` 정사각형이 있습니다.

- 입력:
  ```js
  [
    [1, 2, 3],
    [4, 5, 6],
    [7, 8, 9]
  ]
  ```
  출력: `1`
  - 모든 칸이 서로 다르므로 크기 1보다 큰 같은 숫자 정사각형은 없습니다.

## 힌트
- 어떤 칸을 **정사각형의 오른쪽 아래 끝**이라고 생각해 보세요.
- 그 칸에서 큰 정사각형을 만들려면 위, 왼쪽, 왼쪽 위에도 같은 숫자가 이어져 있어야 합니다.
- 세 방향에서 만들 수 있었던 정사각형 크기 중 가장 작은 값만큼만 현재 칸에서 확장할 수 있습니다.

## 해설
이 문제는 모든 시작점과 모든 크기를 직접 검사하면 매우 느립니다.

예를 들어 각 칸마다 가능한 모든 정사각형을 확인하면,
정사각형 내부를 계속 다시 훑게 되어 큰 입력에서 비효율적입니다.

여기서 중요한 관찰은 다음과 같습니다.

`dp[r][c]`를 **칸 `(r, c)`를 오른쪽 아래 꼭짓점으로 하는, 같은 숫자만으로 이루어진 가장 큰 정사각형의 한 변 길이**라고 정의해 봅시다.

### 1) 기본값
맨 윗줄이나 맨 왼쪽 줄에서는 더 큰 정사각형을 만들 수 없으므로,
각 칸 자체만으로 길이 `1`의 정사각형이 됩니다.

즉, 모든 칸의 기본값은 `1`입니다.

### 2) 점화식
현재 칸이 `(r, c)`이고 값이 `board[r][c]`라고 할 때,
아래 세 칸의 숫자가 모두 현재 칸과 같아야 현재 칸에서 더 큰 정사각형을 만들 수 있습니다.

- 위쪽: `board[r - 1][c]`
- 왼쪽: `board[r][c - 1]`
- 왼쪽 위 대각선: `board[r - 1][c - 1]`

이 셋 중 하나라도 값이 다르면 현재 칸에서 만들 수 있는 최대 정사각형 크기는 `1`입니다.

반대로 세 칸의 숫자가 모두 현재 칸과 같다면,
세 방향에서 이미 만들 수 있었던 정사각형의 크기 중 **가장 작은 값**까지만 안전하게 확장할 수 있습니다.

따라서 점화식은 다음과 같습니다.

```js
if (
  board[r][c] === board[r - 1][c] &&
  board[r][c] === board[r][c - 1] &&
  board[r][c] === board[r - 1][c - 1]
) {
  dp[r][c] = Math.min(dp[r - 1][c], dp[r][c - 1], dp[r - 1][c - 1]) + 1;
} else {
  dp[r][c] = 1;
}
```

왜 `min(...) + 1`일까요?

현재 칸 주변 셋이 모두 같은 숫자여도,
세 방향 중 하나가 길이 2까지만 가능한데 다른 쪽이 길이 5까지 가능하다고 해서 길이 6 정사각형을 만들 수는 없습니다.
중간 어딘가가 비어 있을 수 있기 때문입니다.

그래서 **세 방향 중 가장 작은 정사각형 크기**만큼만 믿고 확장해야 합니다.

### 3) 예시로 보기
다음 보드를 생각해 봅시다.

```js
[
  [1, 2, 2, 2],
  [3, 2, 2, 2],
  [4, 2, 2, 2],
  [5, 6, 7, 8]
]
```

값이 `2`인 구역에서 오른쪽 아래로 내려가며 `dp`를 채우면,
오른쪽 위 `3 x 3` 블록의 마지막 칸에서 길이 `3`이 만들어집니다.

### 4) 복잡도
- 행 수를 `R`, 열 수를 `C`라고 하면
- 모든 칸을 한 번씩만 보므로 시간 복잡도는 `O(R * C)`
- 별도 `dp` 배열을 쓰면 공간 복잡도도 `O(R * C)`입니다.

이 문제를 통해 익힐 포인트는,
**2차원 DP에서 어떤 칸을 끝점으로 정의하고 주변 상태를 이용해 확장하는 방식**입니다.
정사각형의 내부를 매번 다시 검사하지 않고도,
이미 계산한 작은 답들을 이용해 큰 답을 빠르게 만들 수 있습니다.
