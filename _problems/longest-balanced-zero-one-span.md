이진 배열 `bits`가 주어질 때, `0`과 `1`의 개수가 같은 가장 긴 연속 구간의 길이를 반환하는 `solution` 함수를 작성하세요.

## 제한사항
- `1 <= bits.length <= 100000`
- `bits[i]`는 `0` 또는 `1`입니다.
- 반환값은 조건을 만족하는 가장 긴 연속 구간의 길이입니다.
- 조건을 만족하는 구간이 없으면 `0`을 반환합니다.

## 예시
- 입력: `[0, 1]` → 출력: `2`
- 입력: `[0, 1, 0, 0, 1, 1, 0]` → 출력: `6`
  - 예를 들어 `[1, 0, 0, 1, 1, 0]` 구간에는 `0`이 3개, `1`이 3개 있습니다.
- 입력: `[1, 1, 1, 1]` → 출력: `0`
- 입력: `[0, 0, 1, 1, 0]` → 출력: `4`

## 힌트
- `0`을 `-1`, `1`을 `+1`로 바꿔서 생각해 보세요.
- 어떤 구간의 합이 `0`이면, 그 구간 안의 `0`과 `1` 개수가 같다고 볼 수 있습니다.
- 같은 누적합이 처음 나온 위치를 기록해 두면 가장 긴 길이를 빠르게 계산할 수 있습니다.

## 해설
이 문제는 단순히 모든 구간을 확인하면 `O(n^2)`이 되어 비효율적입니다. 대신 **누적합(prefix sum)** 아이디어를 사용하면 한 번의 순회로 해결할 수 있습니다.

핵심 관찰은 다음과 같습니다.

- `0`을 `-1`로 본다
- `1`을 `+1`로 본다

그러면 어떤 연속 구간에서 `0`과 `1`의 개수가 같다는 뜻은, 그 구간의 합이 `0`이라는 뜻과 같습니다.

예를 들어 `[0, 1, 0, 1]`를 `[-1, +1, -1, +1]`로 보면 전체 합은 `0`입니다.
즉, 이 구간은 `0`과 `1`의 개수가 같습니다.

이제 누적합을 왼쪽부터 계산해 봅시다.

- 같은 누적합이 서로 다른 두 위치에서 다시 등장했다면
- 그 사이 구간의 합은 `0`입니다.

예를 들어 누적합이 인덱스 `i`와 `j`에서 모두 `2`였다면,
`i + 1`부터 `j`까지의 합은 `0`이 됩니다.
즉 그 구간은 균형 잡힌 구간입니다.

### 풀이 방법
1. 누적합 `sum`을 0으로 시작합니다.
2. `0`을 만나면 `sum -= 1`, `1`을 만나면 `sum += 1` 합니다.
3. 각 누적합이 **처음 등장한 인덱스**를 `Map`에 저장합니다.
4. 같은 누적합이 다시 나오면, 처음 등장한 위치부터 현재 위치까지의 길이를 계산합니다.
5. 그 길이의 최댓값을 답으로 갱신합니다.

### 왜 처음 등장한 위치만 저장할까?
같은 누적합이 여러 번 나온다면, 가장 앞의 위치와 현재 위치를 연결하는 구간이 가장 길기 때문입니다.
그래서 각 누적합에 대해 **첫 등장 인덱스만 저장**하는 것이 중요합니다.

### 시작 전에 `sum = 0`의 위치를 `-1`로 넣는 이유
배열 맨 앞부터 균형 잡힌 구간이 시작할 수도 있기 때문입니다.
예를 들어 `[0, 1]`은 인덱스 `0`부터 `1`까지가 바로 정답입니다.
이 경우를 자연스럽게 처리하려면 `sum = 0`이 인덱스 `-1`에서 이미 등장했다고 가정하면 됩니다.

### 시간 복잡도
- 배열을 한 번만 순회하므로 `O(n)`
- `Map`에 누적합을 저장하므로 추가 공간은 `O(n)`

이 문제의 핵심은 **배열의 원소 자체를 직접 세는 대신, 누적합이 다시 나타나는 패턴으로 구간 조건을 바꿔 보는 감각**입니다.
