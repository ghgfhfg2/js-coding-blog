현재 시각과 이동할 분 수가 주어질 때, 최종 시각을 구하는 `solution` 함수를 작성하세요.

## 문제 설명
정수 `hour`, `minute`, `diffMinutes`가 주어집니다.

- `hour`와 `minute`는 현재 시각을 나타냅니다.
- `diffMinutes`는 앞으로 또는 뒤로 이동할 분 수입니다.
  - 양수면 미래로 이동합니다.
  - 음수면 과거로 이동합니다.

이동 후의 시각을 **`[hour, minute]` 형태의 배열**로 반환하세요.
시계는 24시간제를 사용하며, 자정을 지나면 다시 `0시 0분`부터 시작합니다.
이전 날로 돌아가는 경우도 같은 방식으로 처리합니다.

예를 들어 `23시 50분`에서 `20분` 앞으로 이동하면 다음 날 `0시 10분`이 됩니다.
따라서 반환값은 `[0, 10]`입니다.

## 제한사항
- `hour`는 0 이상 23 이하의 정수입니다.
- `minute`는 0 이상 59 이하의 정수입니다.
- `diffMinutes`는 -100,000 이상 100,000 이하의 정수입니다.
- 반환값은 `[최종 시, 최종 분]` 형태의 길이 2 배열입니다.
- 최종 시각은 항상 24시간제 범위로 정규화해야 합니다.

## 예시
- 입력: `23, 50, 20` → 출력: `[0, 10]`
- 입력: `0, 10, -20` → 출력: `[23, 50]`
- 입력: `12, 0, 1440` → 출력: `[12, 0]`
- 입력: `5, 30, -330` → 출력: `[0, 0]`

## 힌트
- `hour * 60 + minute`로 현재 시각을 총 분으로 바꿀 수 있습니다.
- 하루는 `24 * 60 = 1440`분입니다.
- 음수가 나올 수 있으니, 나머지 계산 뒤 한 번 더 보정하는 방식을 떠올려 보세요.

## 해설
이 문제의 핵심은 **시각을 총 분으로 바꿔서 계산한 뒤 다시 시와 분으로 나누는 것**입니다.

1. 현재 시각을 `total = hour * 60 + minute`로 바꿉니다.
2. 여기에 `diffMinutes`를 더합니다.
3. 하루는 1440분이므로, `total`을 `0` 이상 `1439` 이하 범위로 정규화합니다.
   - JavaScript에서는 음수 `%` 결과도 음수가 될 수 있으므로 `((total % 1440) + 1440) % 1440`처럼 처리하면 안전합니다.
4. 정규화된 총 분에서:
   - 시 = `Math.floor(total / 60)`
   - 분 = `total % 60`
5. `[시, 분]`을 반환합니다.

예를 들어 `0시 10분`에서 `-20분` 이동하면:

- 현재 총 분: `10`
- 이동 후 총 분: `-10`
- 1440분 기준 정규화: `1430`
- `1430`분은 `23시 50분`

따라서 정답은 `[23, 50]`입니다.

이 문제는 조건문으로 자정 전후를 여러 번 나누기보다, **총 분 변환 + 모듈러 정규화**로 간단하게 해결하는 구현 문제입니다.
