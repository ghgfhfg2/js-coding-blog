체크포인트 기록이 주어질 때, 같은 이름이 다시 등장하는 간격 중 가장 짧은 값을 구하는 `solution` 함수를 작성하세요.

## 문제 설명
문자열 배열 `checkpoints`가 주어집니다.
배열의 각 원소는 방문한 체크포인트 이름을 순서대로 나타냅니다.

같은 체크포인트 이름이 여러 번 등장할 수 있습니다.
이때 **같은 이름이 다시 등장하는 두 위치의 인덱스 차이**를 그 체크포인트의 재등장 간격이라고 하겠습니다.

배열 전체에서 만들 수 있는 재등장 간격들 중 **가장 짧은 값**을 반환하세요.
같은 이름이 한 번도 다시 나오지 않으면 `-1`을 반환합니다.

예를 들어 `checkpoints = ["A", "B", "C", "A", "D", "B"]`라면:

- `"A"`는 인덱스 `0`과 `3`에서 다시 등장하므로 간격은 `3`
- `"B"`는 인덱스 `1`과 `5`에서 다시 등장하므로 간격은 `4`

따라서 가장 짧은 재등장 간격은 `3`입니다.

## 제한사항
- `checkpoints`의 길이는 1 이상 100,000 이하입니다.
- 각 원소는 길이 1 이상 20 이하의 문자열입니다.
- 문자열은 대소문자를 구분합니다.
- 반환값은 가장 짧은 재등장 간격을 나타내는 정수이며, 재등장이 없으면 `-1`입니다.

## 예시
- 입력: `["A", "B", "C", "A", "D", "B"]` → 출력: `3`
- 입력: `["red", "blue", "green", "blue", "red"]` → 출력: `2`
- 입력: `["x", "y", "z"]` → 출력: `-1`
- 입력: `["home", "home", "park", "home"]` → 출력: `1`

## 힌트
- 어떤 이름을 마지막으로 본 위치만 알고 있어도 새로운 재등장 간격을 바로 계산할 수 있습니다.
- 모든 쌍을 직접 비교하면 너무 느릴 수 있습니다.

## 해설
이 문제의 핵심은 **각 체크포인트를 마지막으로 본 인덱스를 기록하는 것**입니다.

배열을 왼쪽부터 순회하면서:

1. 현재 이름이 처음 등장했다면 현재 인덱스를 저장합니다.
2. 이미 등장했던 이름이라면,
   - 현재 인덱스에서 마지막 등장 인덱스를 빼서 간격을 구합니다.
   - 지금까지의 최솟값과 비교해 더 작은 값을 남깁니다.
   - 마지막 등장 인덱스를 현재 위치로 갱신합니다.

이렇게 하면 각 원소를 한 번씩만 확인하므로 효율적으로 풀 수 있습니다.

예를 들어 `checkpoints = ["red", "blue", "green", "blue", "red"]`를 보겠습니다.

- `red`를 인덱스 `0`에 기록
- `blue`를 인덱스 `1`에 기록
- `green`을 인덱스 `2`에 기록
- 다시 `blue`를 인덱스 `3`에서 발견 → 간격은 `3 - 1 = 2`
- 다시 `red`를 인덱스 `4`에서 발견 → 간격은 `4 - 0 = 4`

가장 짧은 값은 `2`이므로 정답은 `2`입니다.

이 문제는 중첩 반복문보다 **해시 맵(Map/Object)으로 최근 위치를 추적하는 방식**이 자연스럽고 효율적인 대표적인 패턴입니다.
