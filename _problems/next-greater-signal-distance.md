정수 배열 `signals`가 주어질 때, 각 위치에서 **오른쪽으로 가며 자신보다 큰 값이 처음 나타나는 위치까지의 칸 수**를 구해 배열로 반환하는 `solution` 함수를 작성하세요.

오른쪽에 자신보다 큰 값이 끝까지 없으면 해당 위치의 값은 `0`입니다.

## 제한사항
- `1 <= signals.length <= 100000`
- `signals[i]`는 정수입니다.
- 같은 값은 **더 큰 값으로 취급하지 않습니다.**
- 반환 배열의 길이는 `signals.length`와 같아야 합니다.

## 예시
- 입력: `[3, 1, 4, 2, 5]` → 출력: `[2, 1, 2, 1, 0]`
  - `3`은 두 칸 뒤의 `4`를 처음 만납니다.
  - `1`은 한 칸 뒤의 `4`를 처음 만납니다.
  - `5`의 오른쪽에는 더 큰 값이 없으므로 `0`입니다.
- 입력: `[5, 4, 3, 2, 1]` → 출력: `[0, 0, 0, 0, 0]`
- 입력: `[1, 3, 3, 2, 6]` → 출력: `[1, 3, 2, 1, 0]`
  - 두 번째 `3`은 마지막 `6`을 처음 만납니다.
  - 첫 번째 `3`도 오른쪽에서 처음 만나는 더 큰 값은 `6`이므로 거리 `3`입니다.

## 힌트
- 각 위치마다 오른쪽을 끝까지 다시 보면 너무 느립니다.
- 아직 더 큰 값을 못 만난 인덱스들을 어떤 구조에 모아 두고, 현재 값이 그 답이 되는 순간 한꺼번에 처리할 수 있습니다.
- 스택에는 값 자체보다 **인덱스**를 넣는 편이 거리 계산에 유리합니다.

## 해설
이 문제를 각 위치마다 오른쪽으로 끝까지 탐색하면 최악의 경우 `O(n^2)`가 됩니다.

핵심은 **아직 자신보다 큰 값을 만나지 못한 위치들**을 관리하는 것입니다. 이때 **단조 감소 스택(monotonic decreasing stack)** 을 사용하면 한 번의 순회로 해결할 수 있습니다.

풀이 흐름은 다음과 같습니다.

1. 정답 배열을 전부 `0`으로 초기화합니다.
2. 스택에는 아직 답을 못 찾은 인덱스를 넣습니다.
3. 왼쪽부터 배열을 순회하면서 현재 값 `signals[i]`를 봅니다.
4. 스택의 맨 위 인덱스 `j`에 대해 `signals[i] > signals[j]`라면,
   - `i`가 `j`의 오른쪽에서 처음 만나는 더 큰 값입니다.
   - 따라서 정답은 `i - j`입니다.
   - 이 과정을 더 이상 조건이 깨질 때까지 반복합니다.
5. 현재 인덱스 `i`를 스택에 넣습니다.
6. 순회가 끝난 뒤에도 스택에 남아 있는 인덱스는 오른쪽에 더 큰 값이 없으므로 그대로 `0`입니다.

예를 들어 `[3, 1, 4, 2, 5]`를 보면:

- `3`을 스택에 넣음
- `1`은 `3`보다 크지 않으니 스택에 넣음
- `4`를 만나면 `1`, `3` 모두의 첫 더 큰 값이 됨
  - `1`의 답은 `1`
  - `3`의 답은 `2`
- `2`는 스택에 넣음
- `5`를 만나면 `2`, `4`의 답을 채움

이 방식에서는 각 인덱스가 스택에 **한 번 들어가고 한 번만 나옵니다.**
따라서 전체 시간 복잡도는 `O(n)`입니다.

이 문제의 포인트는 단순 비교를 반복하는 대신, **아직 해결되지 않은 위치를 스택으로 미뤄 두었다가 현재 값으로 한꺼번에 처리하는 감각**입니다.
