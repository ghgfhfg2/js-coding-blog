정수 배열 `nums`와 정수 `k`가 주어질 때, **배열에서 k번째로 작은 수**를 반환하는 `solution` 함수를 작성하세요.

같은 값이 여러 번 나오면 **중복도 각각 따로 순서에 포함**합니다.
예를 들어 `[1, 2, 2, 5]`에서 3번째로 작은 수는 `2`입니다.

## 제한사항
- `1 <= nums.length <= 100000`
- `-1000000000 <= nums[i] <= 1000000000`
- `1 <= k <= nums.length`
- 중복값이 있을 수 있습니다.
- 반환값은 `nums`를 오름차순으로 나열했을 때 k번째 위치의 값입니다.

## 예시
- 입력: `([7, 2, 5, 3, 9], 2)` → 출력: `3`
- 입력: `([4, 4, 1, 6, 2], 3)` → 출력: `4`
- 입력: `([-5, -1, -3, -2], 1)` → 출력: `-5`
- 입력: `([10], 1)` → 출력: `10`

## 힌트
- 배열 전체를 정렬하면 답을 구할 수는 있지만, 꼭 모든 값을 완전히 정렬할 필요는 없습니다.
- 지금까지 본 값들 중 **가장 작은 k개만 유지**하면 충분합니다.
- 그 k개 안에서 가장 큰 값이 바로 현재의 k번째 작은 수 후보입니다.

## 해설
이 문제는 단순 정렬로도 풀 수 있습니다.
오름차순 정렬 후 `k - 1`번째 값을 반환하면 되기 때문입니다.

하지만 알고리즘 관점에서는 **필요한 정보만 유지하는 방법**을 익히는 것이 더 중요합니다.
여기서는 **크기가 k인 최대 힙(max heap)** 을 사용하는 풀이가 핵심입니다.

### 왜 최대 힙인가?
우리가 알고 싶은 것은 "전체에서 가장 작은 k개"입니다.
이 k개만 잘 유지하고 있다면, 그중 **가장 큰 값**이 바로 k번째 작은 수입니다.

최대 힙을 쓰면 루트가 항상 가장 큰 값이므로,
현재 저장된 k개 후보 중 가장 큰 값을 빠르게 확인하고 교체할 수 있습니다.

### 풀이 흐름
1. 배열을 왼쪽부터 하나씩 봅니다.
2. 최대 힙의 크기가 아직 `k`보다 작다면 현재 값을 그냥 넣습니다.
3. 힙의 크기가 이미 `k`라면:
   - 현재 값이 힙의 루트(= 후보들 중 가장 큰 값)보다 작을 때만
     루트를 빼고 현재 값을 넣습니다.
   - 그렇지 않으면 현재 값은 가장 작은 k개에 들어갈 수 없으므로 버립니다.
4. 순회가 끝난 뒤 힙의 루트가 k번째 작은 수입니다.

### 예시로 이해하기
`nums = [7, 2, 5, 3, 9]`, `k = 2`라고 해봅시다.

- `7` 삽입 → 힙: `[7]`
- `2` 삽입 → 힙: `[7, 2]`
- `5` 확인 → 힙의 루트는 `7`, `5 < 7`이므로 `7` 제거 후 `5` 삽입 → 힙: `[5, 2]`
- `3` 확인 → 루트는 `5`, `3 < 5`이므로 교체 → 힙: `[3, 2]`
- `9` 확인 → `9 < 3`이 아니므로 버림

최종적으로 힙에는 가장 작은 두 수인 `2`, `3`만 남고,
그중 큰 값 `3`이 2번째로 작은 수입니다.

### 왜 이 풀이가 맞는가?
순회 중 언제나 힙에는 **지금까지 본 값 중 가장 작은 k개만** 남도록 유지합니다.

- 힙 크기가 `k`보다 작을 때는 아직 후보를 채우는 단계입니다.
- 힙이 꽉 찬 뒤에는, 루트보다 작은 값만 들어올 자격이 있습니다.
  루트보다 크거나 같은 값은 현재 가장 작은 k개 안에 들어갈 수 없습니다.

따라서 순회가 끝나면 힙은 전체 배열에서 가장 작은 k개를 정확히 담고 있고,
그중 가장 큰 값이 바로 전체 기준 k번째 작은 수가 됩니다.

### 시간 복잡도
- 각 원소를 한 번씩 확인: `O(n)`
- 힙 연산 1회: `O(log k)`
- 전체 시간 복잡도: `O(n log k)`

배열 전체를 정렬하는 `O(n log n)`보다,
특히 `k`가 작을 때 더 효율적으로 동작할 수 있습니다.

이 문제의 포인트는 정렬 자체가 아니라, **정답에 필요한 후보만 남기도록 자료구조를 설계하는 감각**입니다.
