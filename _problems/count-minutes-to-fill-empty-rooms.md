격자 모양 방에서 이미 채워진 칸들이 매 분마다 상하좌우로 퍼질 때, 모든 빈 방이 채워지는 최소 시간을 구하는 `solution` 함수를 작성하세요.

## 문제 설명
2차원 정수 배열 `rooms`가 주어집니다.

각 칸의 의미는 다음과 같습니다.

- `1`: 이미 채워진 방
- `0`: 아직 비어 있는 방
- `-1`: 벽이 있어 퍼질 수 없는 칸

매 분마다 채워진 방은 **상하좌우로 인접한 빈 방(`0`)** 을 동시에 채웁니다.
모든 빈 방이 채워질 수 있다면 걸리는 **최소 분 수**를 반환하세요.

만약 어떤 빈 방은 끝까지 채워질 수 없다면 `-1`을 반환하세요.
이미 처음부터 빈 방이 하나도 없다면 `0`을 반환합니다.

## 제한사항
- `1 <= rooms.length <= 100`
- `1 <= rooms[i].length <= 100`
- `rooms[i][j]`는 `-1`, `0`, `1` 중 하나입니다.
- 퍼짐은 상하좌우 네 방향으로만 일어납니다.
- 입력 격자의 행 길이는 모두 같습니다.

## 예시
- 입력:
  ```js
  [[1,0,0],[0,-1,0],[0,0,0]]
  ```
  출력: `4`

- 입력:
  ```js
  [[1,1,0],[0,0,0]]
  ```
  출력: `2`

- 입력:
  ```js
  [[0,0],[0,0]]
  ```
  출력: `-1`

## 힌트
- 한 칸에서만 시작하는 BFS가 아니라, **처음부터 채워진 모든 칸을 동시에 시작점**으로 둬야 합니다.
- 어떤 분에 새로 채워진 칸들은 그다음 분에 다시 퍼질 수 있습니다.
- 큐에 `(행, 열, 현재까지 걸린 시간)`을 넣거나, 레벨 단위로 큐 크기를 나눠 처리할 수 있습니다.
- 처음에 빈 방 개수를 세어 두면 마지막 판정이 쉬워집니다.

## 해설
이 문제는 격자에서 퍼져 나가는 시간을 구하는 전형적인 **multi-source BFS** 문제입니다.

핵심은 시작점이 하나가 아니라는 점입니다.
이미 채워진 칸 `1`이 여러 개라면, 실제 퍼짐은 그 모든 위치에서 **동시에** 시작됩니다.
따라서 BFS 큐를 만들 때 `1`인 칸들을 전부 먼저 넣어야 합니다.

### 1. 왜 BFS가 맞을까?
한 번 퍼질 때마다 시간은 정확히 `1분` 늘어납니다.
즉, 어떤 빈 방이 처음 채워지는 순간은
가장 가까운 시작점으로부터 도달하는 최소 시간과 같습니다.
이런 “최소 단계 수” 문제는 BFS가 잘 맞습니다.

### 2. 초기 상태 준비
먼저 격자를 한 번 순회하면서 다음을 기록합니다.

- 이미 채워진 칸(`1`)의 위치들 → BFS 시작점으로 큐에 모두 삽입
- 빈 방(`0`)의 개수 → `emptyCount`

여기서 `emptyCount`가 처음부터 `0`이면, 더 퍼질 필요가 없으므로 정답은 바로 `0`입니다.

### 3. BFS 진행
큐에서 하나씩 꺼내며 상하좌우 네 방향을 확인합니다.

- 범위를 벗어나면 무시
- 벽(`-1`)이면 무시
- 이미 채워졌거나 방문한 칸이면 무시
- 빈 방(`0`)이면 새로 채웁니다

새로 채운 칸은:

1. 값을 `1`로 바꾸거나 방문 처리하고
2. `emptyCount`를 1 줄이고
3. 현재 시간 + 1과 함께 큐에 넣습니다

이 과정을 반복하면, 나중에 큐에서 꺼낸 시간값 중 가장 큰 값이 모든 퍼짐이 끝난 시점이 됩니다.

### 4. 끝까지 못 채우는 경우
BFS가 끝난 뒤에도 `emptyCount > 0`이라면,
벽 때문에 도달할 수 없는 빈 방이 남아 있다는 뜻입니다.
이 경우 `-1`을 반환하면 됩니다.

### 5. 예시 흐름
`[[1,0,0],[0,-1,0],[0,0,0]]`을 보겠습니다.

처음 시작점은 `(0, 0)` 하나입니다.

- 0분: `(0,0)`
- 1분: `(0,1)`, `(1,0)`
- 2분: `(0,2)`, `(2,0)`
- 3분: `(1,2)`, `(2,1)`
- 4분: `(2,2)`

모든 빈 방이 채워지는 데 `4분`이 걸립니다.

### 6. 시간 복잡도
각 칸은 최대 한 번만 큐에 들어갑니다.
따라서 시간 복잡도는 `O(R * C)`이고,
추가 공간도 큐 때문에 `O(R * C)`입니다.

이 문제의 학습 포인트는 **여러 시작점에서 동시에 퍼지는 상황을 BFS 한 번으로 처리하는 감각**입니다.
한 시작점씩 따로 BFS를 돌리면 비효율적이고, 동시에 시작해야 실제 시간 흐름도 올바르게 반영됩니다.
