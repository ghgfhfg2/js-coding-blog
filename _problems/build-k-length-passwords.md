서로 다른 숫자들이 담긴 배열 `digits`와 정수 `k`가 주어질 때, **숫자를 중복 없이 사용해서 만들 수 있는 길이 `k`의 모든 비밀번호**를 **사전순**으로 반환하는 `solution` 함수를 작성하세요.

각 비밀번호는 문자열로 반환해야 하며, 숫자의 순서가 다르면 서로 다른 비밀번호로 봅니다.

예를 들어 `digits = [3, 1, 2]`, `k = 2`이면 만들 수 있는 비밀번호는 `"12"`, `"13"`, `"21"`, `"23"`, `"31"`, `"32"`입니다.

## 제한사항
- `1 <= digits.length <= 8`
- `digits`의 각 원소는 `0` 이상 `9` 이하의 정수입니다.
- `digits` 안의 값은 서로 다릅니다.
- `1 <= k <= 8`
- `k`가 `digits.length`보다 크면 만들 수 있는 비밀번호가 없으므로 빈 배열 `[]`를 반환합니다.
- 반환 배열은 **사전순 오름차순**이어야 합니다.
- `0`으로 시작하는 비밀번호도 문자열이라면 정상적인 정답입니다.

## 예시
- 입력: `digits = [3, 1, 2]`, `k = 2` → 출력: `["12", "13", "21", "23", "31", "32"]`
- 입력: `digits = [0, 7, 5]`, `k = 2` → 출력: `["05", "07", "50", "57", "70", "75"]`
- 입력: `digits = [9, 4]`, `k = 3` → 출력: `[]`
  - 사용할 수 있는 숫자 개수보다 더 긴 비밀번호는 만들 수 없습니다.
- 입력: `digits = [8, 1, 6]`, `k = 1` → 출력: `["1", "6", "8"]`

## 힌트
- 먼저 숫자를 정렬해 두면, 탐색하면서 만든 결과도 사전순으로 관리하기 쉬워집니다.
- 현재 비밀번호에 어떤 숫자들을 이미 사용했는지 기록해야 합니다.
- 길이가 `k`가 될 때마다 결과 배열에 추가하고, 다음 경우의 수를 위해 직전 선택을 되돌리는 **백트래킹**을 떠올려 보세요.

## 해설
이 문제의 핵심은 **현재까지 만든 비밀번호를 한 글자씩 확장하면서 가능한 모든 경우를 빠짐없이 탐색하되, 이미 사용한 숫자는 다시 쓰지 않는 것**입니다.

이런 유형은 **DFS(깊이 우선 탐색) + 백트래킹**으로 풀기 좋습니다.

### 1) 왜 백트래킹이 잘 맞을까?
길이 `k`의 비밀번호를 만들려면,
- 첫 번째 자리에 어떤 숫자를 넣을지 고르고
- 그다음 남은 숫자 중 하나를 두 번째 자리에 넣고
- 이렇게 길이가 `k`가 될 때까지 선택을 이어 가면 됩니다.

각 단계에서 선택지가 여러 개 갈라지므로 트리처럼 생각할 수 있습니다.
이때 한 경로를 끝까지 내려가 비밀번호 하나를 완성한 뒤, 다시 이전 상태로 돌아와 다른 선택을 이어 가는 방식이 바로 백트래킹입니다.

### 2) 사전순은 어떻게 맞출까?
입력 `digits`가 정렬되어 있다는 보장이 없으므로 먼저 오름차순으로 정렬합니다.
그 뒤 DFS에서 항상 작은 숫자부터 차례대로 선택하면, 완성되는 비밀번호도 자연스럽게 사전순으로 쌓입니다.

예를 들어 `[3, 1, 2]`를 정렬하면 `[1, 2, 3]`이 되고,
- `1`로 시작하는 비밀번호들
- `2`로 시작하는 비밀번호들
- `3`로 시작하는 비밀번호들
순서로 생성되므로 결과가 사전순이 됩니다.

### 3) 어떤 상태를 관리해야 하나?
탐색 중에는 보통 아래 두 가지 상태가 필요합니다.

- `path`: 지금까지 고른 숫자들
- `used`: 각 숫자를 이미 사용했는지 여부

탐색 흐름은 다음과 같습니다.

1. `path.length === k`가 되면 비밀번호 하나가 완성된 것입니다.
2. `path`를 문자열로 합쳐 결과 배열에 넣습니다.
3. 아직 길이가 부족하면, 모든 숫자를 보면서 아직 사용하지 않은 숫자만 선택합니다.
4. 숫자를 선택한 뒤 재귀 호출을 합니다.
5. 호출이 끝나면 방금 선택한 숫자를 다시 빼고 `used`도 되돌립니다.

이 마지막 되돌리기 과정이 백트래킹의 핵심입니다.

### 4) 예시로 흐름 보기
`digits = [3, 1, 2]`, `k = 2`라면 정렬 후 `[1, 2, 3]`입니다.

- `1` 선택
  - `2` 선택 → `"12"`
  - `3` 선택 → `"13"`
- `2` 선택
  - `1` 선택 → `"21"`
  - `3` 선택 → `"23"`
- `3` 선택
  - `1` 선택 → `"31"`
  - `2` 선택 → `"32"`

결과는 `["12", "13", "21", "23", "31", "32"]`입니다.

### 5) 예외 상황
`k > digits.length`이면 서로 다른 숫자를 중복 없이 써야 한다는 조건 때문에 아예 만들 수 있는 비밀번호가 없습니다.
이 경우는 탐색을 시작하기 전에 바로 `[]`를 반환하면 됩니다.

또 `0`이 들어 있어도 문자열 비밀번호이므로 `"05"` 같은 값은 정상적인 결과입니다.
숫자로 바꾸면 앞의 0이 사라질 수 있으니, 끝까지 문자열로 다루는 것이 안전합니다.

### 6) 시간 복잡도
가능한 비밀번호 개수는 순열 개수와 같아서 대략

- `P(n, k) = n * (n - 1) * ... * (n - k + 1)`

개입니다.
각 결과를 만드는 데 길이 `k`만큼의 작업이 들어가므로, 전체 시간은 생성해야 하는 결과 수에 비례합니다.

이 문제의 포인트는 단순히 모든 경우를 만드는 것이 아니라, **정렬로 출력 순서를 제어하고, 사용 여부를 관리하며, 탐색 후 상태를 정확히 되돌리는 백트래킹 감각**을 익히는 데 있습니다.
