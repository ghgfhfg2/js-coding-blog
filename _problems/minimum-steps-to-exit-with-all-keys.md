격자 미로 `grid`가 주어질 때, 시작점 `S`에서 출발해 **격자 안의 모든 소문자 열쇠를 최소 한 번씩 수집한 뒤** 출구 `E`에 도달하는 최소 이동 칸 수를 구하는 `solution` 함수를 작성하세요.

이동은 상하좌우로만 가능하며, 대문자 문 `A`~`Z`는 대응하는 소문자 열쇠 `a`~`z`를 이미 가지고 있을 때만 통과할 수 있습니다. 도달할 수 없으면 `-1`을 반환하세요.

## 제한사항
- `1 <= grid.length <= 30`
- `1 <= grid[i].length <= 30`
- 모든 행의 길이는 같습니다.
- 문자는 `S`, `E`, `.`, `#`, `a`~`f`, `A`~`F` 중 하나입니다.
- `#`는 벽이며 지나갈 수 없습니다.
- 시작점 `S`와 출구 `E`는 정확히 하나씩 존재합니다.
- 열쇠는 최대 6개입니다.
- 반환값은 최소 이동 횟수이며, 불가능하면 `-1`입니다.

## 예시
- 입력:
  ```js
  [
    "S.a",
    ".#A",
    "..E"
  ]
  ```
  출력: `4`
  - `a`를 먼저 줍고, 그 뒤 출구 `E`로 가면 됩니다.

- 입력:
  ```js
  [
    "S..b",
    "##.#",
    "a..B",
    "A..E"
  ]
  ```
  출력: `9`
  - `b`를 먼저 얻은 뒤 아래로 돌아가 `a`까지 챙기고, 마지막에 출구로 이동해야 합니다.

- 입력:
  ```js
  [
    "S#A",
    "a#E",
    "..."
  ]
  ```
  출력: `5`
  - 바로 문 `A`를 통과할 수 없으므로 아래로 돌아 `a`를 얻은 뒤 출구로 가야 합니다.

## 힌트
- 단순히 `(행, 열)`만 방문 처리하면 정답이 틀릴 수 있습니다.
- 같은 칸이라도 **어떤 열쇠들을 이미 모았는지**에 따라 앞으로 갈 수 있는 경로가 달라집니다.
- 따라서 상태를 `위치 + 열쇠 보유 상태`로 확장해 BFS를 해야 합니다.

## 해설
이 문제의 핵심은 **같은 좌표라도 열쇠 상태가 다르면 전혀 다른 상태**라는 점입니다.

예를 들어 어떤 칸에 한 번 도착했더라도,
- 아직 `a` 열쇠가 없는 상태와
- `a` 열쇠를 얻은 뒤 다시 도착한 상태는

이후 열 수 있는 문이 달라지므로 같은 방문으로 취급하면 안 됩니다.

그래서 BFS 상태를 다음처럼 잡아야 합니다.

- 현재 행 `r`
- 현재 열 `c`
- 현재까지 모은 열쇠 집합 `mask`

열쇠는 최대 6개이므로, 비트마스크로 관리할 수 있습니다.
예를 들어:
- `a`를 얻으면 `1 << 0`
- `b`를 얻으면 `1 << 1`
- `c`를 얻으면 `1 << 2`

처럼 표현하면, 현재 가진 열쇠 집합을 정수 하나로 저장할 수 있습니다.

### 풀이 흐름
1. 격자를 순회하며 시작점 `S`와 전체 열쇠 개수를 찾습니다.
2. 모든 열쇠를 모은 상태를 `targetMask`로 만듭니다.
3. BFS 큐에는 `(r, c, mask, dist)`를 넣습니다.
4. 인접 칸으로 이동할 때:
   - 벽 `#`이면 건너뜁니다.
   - 소문자 열쇠면 해당 비트를 켭니다.
   - 대문자 문이면 대응 열쇠 비트가 켜져 있을 때만 통과합니다.
5. 어떤 상태가 `E`에 도착했고 `mask === targetMask`라면, 그때의 거리가 정답입니다.
6. 끝까지 못 찾으면 `-1`입니다.

### 왜 BFS가 맞는가?
모든 이동 비용이 1칸으로 동일하므로, BFS는 상태 그래프에서 최단 거리를 보장합니다.
다만 이 문제의 그래프 정점은 단순한 칸이 아니라 **`(위치, 열쇠 상태)` 조합**입니다.

즉 방문 배열도 `visited[row][col][mask]` 형태가 되어야 합니다.

### 시간 복잡도
격자 크기를 `R x C`, 열쇠 수를 `K`라고 하면 가능한 상태 수는 최대 `R * C * 2^K`입니다.
열쇠가 최대 6개이므로 `2^K <= 64`라서 충분히 관리 가능합니다.

이 문제의 포인트는 미로 탐색 자체보다, **탐색 상태를 확장해서 같은 칸을 여러 의미로 방문할 수 있게 모델링하는 감각**입니다.
