로그인 세션 구간 목록 `sessions`가 주어질 때, **가장 많은 사용자가 동시에 접속 중이었던 시각**과 그때의 **동시 접속 수**를 구하는 `solution` 함수를 작성하세요.

각 세션은 `[start, end]` 형태의 두 정수로 주어지며, 이 세션은 **`start` 시각에는 접속 중이고 `end` 시각에는 이미 종료된 상태**라고 가정합니다. 즉, 각 세션은 **반열린 구간 `[start, end)`** 입니다.

반환값은 `[earliestTime, maxCount]` 형식의 배열입니다.

- `earliestTime`: 최대 동시 접속 수가 **처음** 만들어진 시각
- `maxCount`: 그 시각의 동시 접속 수

## 제한사항
- `1 <= sessions.length <= 100000`
- 각 세션은 `[start, end]` 형태입니다.
- `-10^9 <= start < end <= 10^9`
- 입력 순서는 정렬되어 있지 않을 수 있습니다.
- 같은 시각에 여러 세션이 시작하거나 종료할 수 있습니다.
- 같은 시각에 시작과 종료가 함께 있으면, **종료를 먼저 처리**해야 합니다.

## 예시
- 입력: `[[-3, 2], [0, 4], [1, 5], [6, 8]]` → 출력: `[1, 3]`
  - 시각 `1`부터 세 개 세션이 동시에 열려 있습니다.
- 입력: `[[1, 3], [3, 5], [3, 6], [5, 7]]` → 출력: `[3, 2]`
  - `[1, 3)`은 시각 `3`에서 이미 끝난 상태이므로, 시각 `3`의 동시 접속 수는 `2`입니다.
- 입력: `[[5, 10], [5, 10], [5, 10], [8, 12]]` → 출력: `[5, 3]`
  - 시각 `5`에 최대 동시 접속 수 `3`이 처음 만들어집니다.

## 힌트
- 각 구간을 그대로 비교하기보다, 시작과 종료를 각각 이벤트로 바꿔 생각해 보세요.
- 시작은 `+1`, 종료는 `-1`처럼 표현할 수 있습니다.
- 같은 시각의 이벤트 처리 순서가 결과에 영향을 줍니다.

## 해설
이 문제를 단순하게 풀면, 모든 시각을 하나씩 확인하거나 모든 구간 쌍을 비교해야 해서 비효율적입니다.

핵심은 **구간 전체를 보지 말고, 변화가 일어나는 시각만 보자**는 것입니다.

### 1) 시작과 종료를 이벤트로 바꾸기
세션 `[start, end)` 하나는 다음 두 이벤트로 바꿀 수 있습니다.

- `(start, +1)` : 이 시각부터 접속 수가 1 증가
- `(end, -1)` : 이 시각부터는 이미 종료되므로 접속 수가 1 감소

예를 들어 `[2, 5)`는
- 시각 `2`에서 +1
- 시각 `5`에서 -1
입니다.

### 2) 이벤트를 시간순으로 정렬하기
모든 이벤트를 모아 시간순으로 정렬한 뒤 왼쪽부터 스캔하면, 현재 시각의 동시 접속 수를 계속 갱신할 수 있습니다.

그런데 중요한 함정이 하나 있습니다.

### 3) 같은 시각이면 종료를 먼저 처리해야 함
문제에서 세션은 `[start, end)` 이므로 `end` 시각에는 이미 접속 중이 아닙니다.

따라서 같은 시각 `t`에
- 어떤 세션은 끝나고
- 다른 세션은 시작한다면

**종료(-1)를 먼저 처리한 뒤 시작(+1)을 처리해야** 올바른 동시 접속 수가 됩니다.

예를 들어 `[[1, 3), [3, 5), [3, 6)]`에서는 시각 `3`에서
- `[1, 3)`은 빠지고
- 나머지 두 세션이 들어옵니다.

그래서 시각 `3`의 접속 수는 `3`이 아니라 `2`입니다.

### 4) 최댓값과 최초 시각 기록하기
이벤트를 순서대로 처리하면서 현재 접속 수 `active`를 유지합니다.

- 이벤트를 적용한 뒤의 `active`가 지금까지의 최댓값보다 크면
  - `maxCount = active`
  - `earliestTime = 현재 시각`
- 같다면 갱신하지 않습니다.
  - 문제는 **최대가 처음 만들어진 시각**을 원하기 때문입니다.

### 5) 왜 이 방식이 효율적인가?
세션이 `n`개라면 이벤트는 `2n`개뿐입니다.

- 이벤트 생성: `O(n)`
- 정렬: `O(n log n)`
- 스캔: `O(n)`

따라서 전체 시간 복잡도는 `O(n log n)`입니다.

이 문제의 포인트는 구간 문제를 그대로 다루지 않고, **시작/종료 이벤트로 변환한 뒤 정렬해서 훑는 sweep line 관점**으로 바꾸는 데 있습니다.
