정수 배열 `nums`가 주어질 때, **왼쪽부터 읽어 가며 가장 먼저 다시 등장하는 숫자**를 반환하는 `solution` 함수를 작성하세요.

반복해서 나타나는 숫자가 없다면 `-1`을 반환하세요.

## 제한사항
- `1 <= nums.length <= 100000`
- `nums[i]`는 정수입니다.
- 음수와 `0`이 포함될 수 있습니다.
- 반환값은 가장 먼저 두 번째 등장까지 확인되는 숫자이며, 없으면 `-1`입니다.

## 예시
- 입력: `[4, 7, 1, 7, 4]` → 출력: `7`
  - `7`이 먼저 다시 등장하므로 정답입니다.
- 입력: `[9, 2, 5, 2, 9]` → 출력: `2`
  - `9`도 반복되지만, `2`의 두 번째 등장이 더 먼저 나옵니다.
- 입력: `[1, 2, 3, 4]` → 출력: `-1`

## 힌트
- 지금까지 본 숫자를 빠르게 확인할 수 있으면 됩니다.
- 배열을 왼쪽부터 한 번만 보면서, 이미 나온 숫자인지 검사해 보세요.
- 처음으로 “이미 본 숫자”가 되는 순간 바로 답을 반환할 수 있습니다.

## 해설
이 문제의 핵심은 **등장 횟수를 끝까지 다 셀 필요 없이, 두 번째로 나타나는 첫 숫자를 즉시 찾는 것**입니다.

예를 들어 `[4, 7, 1, 7, 4]`를 왼쪽부터 보면:

1. `4`는 처음 보므로 기록합니다.
2. `7`도 처음 보므로 기록합니다.
3. `1`도 처음 보므로 기록합니다.
4. 다음 `7`은 이미 본 숫자이므로, 이 순간 바로 정답은 `7`입니다.
5. 뒤에 있는 `4`는 더 나중에 다시 나오므로 볼 필요가 없습니다.

이때 `Set`을 사용하면 어떤 숫자를 이미 봤는지 빠르게 확인할 수 있습니다.

풀이 흐름은 다음과 같습니다.

1. 빈 `Set`을 준비합니다.
2. 배열을 앞에서부터 순회합니다.
3. 현재 숫자가 `Set`에 이미 있으면 그 숫자를 바로 반환합니다.
4. 아직 없다면 `Set`에 추가합니다.
5. 끝까지 반복되는 숫자가 없으면 `-1`을 반환합니다.

이 방법은 배열을 한 번만 보므로 시간 복잡도는 `O(n)`이고, `Set`에 본 숫자들을 저장하므로 추가 공간 복잡도는 `O(n)`입니다.

포인트는 **가장 많이 반복된 숫자**가 아니라, **가장 먼저 다시 나타난 숫자**를 찾는다는 점입니다.
