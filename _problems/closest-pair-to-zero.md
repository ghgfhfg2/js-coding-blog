정수 배열 `nums`가 주어질 때, **합이 0에 가장 가까운 두 수**를 찾아 `[a, b]` 형태로 반환하세요.

반환하는 두 수는 항상 **오름차순**이어야 합니다.

합의 절댓값이 같은 쌍이 여러 개라면 아래 규칙으로 하나를 고릅니다.

1. 첫 번째 수가 더 작은 쌍
2. 첫 번째 수도 같다면 두 번째 수가 더 작은 쌍

## 제한사항
- `2 <= nums.length <= 100000`
- `-10^9 <= nums[i] <= 10^9`
- 배열은 정렬되어 있지 않을 수 있습니다.
- 같은 값이 여러 번 등장할 수 있습니다.
- 반드시 서로 다른 두 인덱스의 수를 골라야 합니다.

## 예시
- 입력: `[-8, -3, 2, 5, 9]` → 출력: `[-3, 2]`
  - 합은 `-1`이고, 절댓값은 `1`입니다.
- 입력: `[-10, -4, -1, 3, 7, 12]` → 출력: `[-4, 3]`
  - 합은 `-1`이고, 절댓값은 `1`이라 가장 가깝습니다.
- 입력: `[1, 4, 6, 9]` → 출력: `[1, 4]`
  - 모두 양수라면 가장 작은 두 수를 고르는 것이 0에 가장 가깝습니다.
- 입력: `[-7, -5, -2]` → 출력: `[-5, -2]`
  - 모두 음수라면 가장 큰 두 수를 고르는 것이 0에 가장 가깝습니다.

## 힌트
- 먼저 배열을 정렬하면, 가장 작은 값은 왼쪽 끝에 가장 큰 값은 오른쪽 끝에 있습니다.
- 두 수의 합이 너무 작다면(음수라면) 더 큰 값을 써야 0에 가까워집니다.
- 반대로 합이 너무 크다면(양수라면) 더 작은 값을 써야 합니다.
- 그래서 양끝에서 시작하는 **투 포인터**가 잘 맞습니다.

## 해설
이 문제를 모든 쌍으로 확인하면 `O(n^2)`이 걸립니다. `n`이 최대 `100000`이므로 너무 느립니다.

핵심은 **정렬 후 양끝에서 좁혀 오기**입니다.

### 1) 왜 정렬이 도움이 되나?
배열을 오름차순으로 정렬하면,
- 왼쪽 포인터 `left`는 작은 수
- 오른쪽 포인터 `right`는 큰 수
를 가리키게 됩니다.

이 상태에서 `nums[left] + nums[right]`를 보면 합이
- **음수**라면 더 큰 값이 필요하므로 `left`를 오른쪽으로 이동
- **양수**라면 더 작은 값이 필요하므로 `right`를 왼쪽으로 이동
하면 됩니다.

이렇게 하면 매번 한쪽 포인터만 움직여도, 0에 더 가까운 후보를 체계적으로 탐색할 수 있습니다.

### 2) 어떤 값을 답으로 저장하나?
현재 후보 `(a, b)`에 대해
- `abs(a + b)`가 더 작으면 갱신
- 절댓값이 같다면 문제의 동률 규칙대로
  - 첫 번째 수가 더 작은 쌍
  - 그래도 같으면 두 번째 수가 더 작은 쌍
을 선택합니다.

정렬된 배열에서 `left < right`를 유지하므로 반환 쌍은 자연스럽게 오름차순입니다.

### 3) 진행 예시
`[-10, -4, -1, 3, 7, 12]`를 보겠습니다.

- 시작: `(-10, 12)` → 합 `2`, 절댓값 `2`
- 합이 양수이므로 `right`를 왼쪽으로 이동
- `(-10, 7)` → 합 `-3`, 절댓값 `3`
- 합이 음수이므로 `left`를 오른쪽으로 이동
- `(-4, 7)` → 합 `3`, 절댓값 `3`
- 합이 양수이므로 `right`를 왼쪽으로 이동
- `(-4, 3)` → 합 `-1`, 절댓값 `1` → 더 좋은 답으로 갱신
- 합이 음수이므로 `left`를 오른쪽으로 이동
- `(-1, 3)` → 합 `2`, 절댓값 `2`

최종 답은 `[-4, 3]`입니다. 이렇게 포인터를 한 칸씩만 움직여도 가능한 좋은 후보를 빠짐없이 압축해서 볼 수 있습니다.

### 4) 알고리즘 정리
1. 배열을 오름차순 정렬합니다.
2. `left = 0`, `right = nums.length - 1`로 둡니다.
3. `left < right` 동안:
   - 현재 쌍을 답 후보와 비교해 필요하면 갱신합니다.
   - 합이 0보다 작으면 `left++`
   - 합이 0보다 크면 `right--`
   - 합이 0이면 더 이상 절댓값이 줄어들 수 없으므로 종료할 수 있습니다.
4. 저장한 최적 쌍을 반환합니다.

### 5) 복잡도
- 정렬: `O(n log n)`
- 투 포인터 스캔: `O(n)`
- 전체: `O(n log n)`

이 문제에서 익혀야 할 감각은 **정렬 + 투 포인터 조합으로 모든 쌍을 직접 보지 않고도 최적 해를 찾는 방법**입니다.
