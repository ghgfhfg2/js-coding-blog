정수 배열 `nums`와 정수 `k`가 주어질 때, 길이가 정확히 `k`인 **모든 연속 구간의 최솟값**을 구한 뒤 그 값들 중 **가장 큰 값**을 반환하는 `solution` 함수를 작성하세요.

예를 들어 `nums = [4, 2, 5, 1, 3]`, `k = 3`이면 가능한 구간은 `[4,2,5]`, `[2,5,1]`, `[5,1,3]`이고 각 구간의 최솟값은 `2`, `1`, `1`이므로 정답은 `2`입니다.

## 제한사항
- `1 <= nums.length <= 100000`
- `-1000000000 <= nums[i] <= 1000000000`
- `1 <= k <= nums.length`
- 반환값은 길이 `k`인 모든 연속 구간의 최솟값들 중 가장 큰 정수입니다.

## 예시
- 입력: `nums = [4, 2, 5, 1, 3]`, `k = 3` → 출력: `2`
  - 구간 최솟값은 `[2, 1, 1]`이고, 이 중 가장 큰 값은 `2`입니다.
- 입력: `nums = [9, 1, 8, 2, 7, 3]`, `k = 2` → 출력: `3`
  - 구간 최솟값은 `[1, 1, 2, 2, 3]`이므로 정답은 `3`입니다.
- 입력: `nums = [5, 4, 3]`, `k = 3` → 출력: `3`
  - 구간이 하나뿐이므로 그 구간의 최솟값 `3`을 그대로 반환합니다.

## 힌트
- 구간마다 최솟값을 새로 구하면 최악의 경우 `O(nk)`가 되어 비효율적입니다.
- 현재 구간에서 **가장 작은 후보들만 남기는 덱**을 유지하면 각 칸을 한 번씩만 처리할 수 있습니다.
- 덱에는 값을 직접 넣기보다 **인덱스**를 넣어야 범위를 벗어난 원소를 쉽게 제거할 수 있습니다.

## 해설
이 문제는 길이 `k`인 모든 구간을 보되, 각 구간의 최솟값을 빠르게 알아내는 것이 핵심입니다.

가장 단순한 방법은 구간이 시작할 때마다 길이 `k`를 전부 훑어서 최솟값을 구하는 것입니다. 하지만 이 방법은 배열 길이가 커지면 너무 느립니다.

여기서 사용할 수 있는 도구가 **단조 증가 덱(monotonic deque)** 입니다.

### 어떻게 덱을 유지할까?
덱에는 현재 구간 안에서 **최솟값 후보가 될 수 있는 인덱스들**만 남깁니다.

새 인덱스 `i`를 처리할 때는 두 가지를 합니다.

1. 덱 맨 앞이 현재 구간 `[i-k+1, i]` 밖으로 나갔다면 제거합니다.
2. 덱 맨 뒤의 값이 `nums[i]`보다 크다면, 그 값은 앞으로 최솟값 후보가 될 일이 없으므로 제거합니다.

이 과정을 거치면 덱은 항상 값이 증가하는 형태를 유지하고, 덱의 맨 앞 인덱스가 현재 구간의 최솟값 위치가 됩니다.

### 답은 어떻게 갱신할까?
인덱스 `i`가 `k-1` 이상이 되는 순간부터 길이 `k`인 구간 하나가 완성됩니다.
그때마다 현재 구간의 최솟값 `nums[deque[0]]`을 확인하고, 지금까지의 답과 비교해 더 큰 값을 남기면 됩니다.

### 왜 빠를까?
각 인덱스는 덱에 **최대 한 번 들어가고 한 번 나옵니다.**
따라서 전체 시간 복잡도는 `O(n)`이고, 추가 공간은 덱에 사용하는 `O(k)` 수준입니다.

즉,
- 구간 최솟값은 덱 앞에서 즉시 확인하고
- 모든 구간을 한 번만 밀면서
- 그 최솟값들 중 가장 큰 값까지 동시에 구할 수 있습니다.
